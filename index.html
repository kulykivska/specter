How it works

Four steps. Zero manual work.

Specter runs automatically. Commit config.yml, push, and never think about it again.

01
Watch Jira
Polls your project every 30 minutes. Uses the Jira changelog API to detect exactly which fields changed.
JQL + changelog
02
Parse content
Extracts summary, description, and all comments. Converts Atlassian Document Format to clean plain text.
ADF parser
03
Generate Gherkin
Sends task content to your chosen LLM. Receives structured Gherkin covering happy path, edge cases, and negative scenarios.
Claude / GPT-4o
04
Save results
Creates a Jira subtask, writes a .feature file, posts to Slack, or all three simultaneously.
Jira · File · Slack

Architecture

Three pluggable layers.
Extend without rewriting.

Every component is an abstract interface. The generator never knows which provider is underneath.

Layer 01

Triggers

Define when and which tasks to process. Polling reads the Jira changelog to detect field-level changes.

  • PollingTrigger - cron, every 30 min
  • ManualTrigger - all tasks or by key
  • watch_fields: summary / description / comment
  • Extend via BaseTrigger
Layer 02

LLM Client

One config line to switch AI providers. All implement the same generate() interface - the rest of the system is unaffected.

  • AnthropicLLMClient
  • OpenAILLMClient
  • AzureOpenAILLMClient
  • Exponential backoff on rate limits
Layer 03

Outputs

Enable any combination simultaneously. Results go exactly where your team needs them.

  • JiraOutput - create / upsert / skip
  • FileOutput - .feature files
  • SlackOutput - post to channel
  • SlackNotification - event alerts
Outputs

Where the Gherkin lands.

Enable multiple outputs at once. All run in parallel for each processed task.

Jira Subtask

Creates "Test Cases: {summary}" as a subtask. Three modes adapt to your team's workflow.

create - always new
upsert - update if exists
skip_if_exists - preserve edits

.feature Files

Saves Gherkin to {issue_key}.feature in a configured path. Works with Cucumber and Behave.

Configurable output path
Directories auto-created

Slack

Posts to your QA channel via Incoming Webhook. Full Gherkin or a summary link - configurable per team.

post_gherkin: true / false
Separate notification channel
LLM Providers

One line to
switch AI providers.

Change llm.provider in config.yml. No code changes required.

Anthropicclaude-sonnet-4Recommended
OpenAIgpt-4o

Add a custom provider in ~20 lines of Python
config.yml
# Switch provider with one line
llm:
  provider: anthropic
  model: claude-sonnet-4-20250514
  temperature: 0

triggers:
  polling:
    enabled: true
    interval_minutes: 35
    watch_fields: [summary, description, comment]

outputs:
  jira:
    enabled: true
    mode: upsert
  slack:
    enabled: true
    channel: "#qa-test-cases"

notifications:
  slack:
    enabled: true
    channel: "#qa-alerts"
    notify_on: [generated, error]
